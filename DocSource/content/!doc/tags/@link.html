@link

@link is not a tag, but something like keyword. GenDoc.exe replaces the "@link:" occurrences to the documentation path from the site corner, e.g. "/jsgui-doc/content/doc". Please don't forget the tailing ":" after "@link".

For example, <a href="@link:/core/data-object.js/Data_Object.html#dobj__obj_data_def">Data_Object.dobj()</a> will produce the following result: Data_Object.dobj()
