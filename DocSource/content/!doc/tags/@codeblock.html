<@codeblock>
Example
<@codeblock> var pow2 = function (item) { return item * item; } // jsgui_util.execute_on_each_simple([1, 2, 3], pow2) ==> [1, 4, 9] jsgui_util.execute_on_each_simple({ a: 1, b: 2, c: 3 }, pow2) ==> [1, 4, 9] </@codeblock> <@codeblock> <@codeblock src="/z_core/data-object/Data_Object.spec.js:field()" />
Result:
Description:

Inserts a code block in the documentation (using PRE and CODE tags, and prettyprint syntax highlight capability form JsDoc).

The code can be placed directly into the <@codeblock> tag. In this case the resulting box will contains a "Static code" badge on the top right.

Also, the code can be read from an unit test output (the test is specified by the src attribute). In this case the top right badge will show the unit test passing state (Passed, Failed, Pending, or Error - if the unit test code was not found).

The src attribute value is the unit test file path, then ":" char, then unit test name. For example: src="/z_core/data-object/Data_Object.spec.js:field()" means: insert code from "field()" unit test from (DocWebSite)\jsgui-doc\content-dev\tests\z_core\data-object\Data_Object.spec.js.html file.
