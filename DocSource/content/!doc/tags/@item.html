<@item> tag

The <@item> tag generates an expandable item block (initially collapsed) for a method, remarks, etc. The generated HTML consists of <dt> and <dd> tags pair.

Example:
<@codeblock> <@item title="field()">

Returns an array containing fields definition.

Attributes:
title

The item title, e.g. "Remarks", "field()", "static dobj(obj-opt, data_def-opt)" etc. Usually the title is a method signature. The first "static " word (if present) will be decorated. Parameters with "-opt" suffix will be decorated.

For example: <@item title="static dobj(obj-opt, data_def-opt)"> will produce the following header:

The <@item> tag adds an HTML id attribute to the generated header code. It allows to refer the item like <a href="filename.html#item_id">. The id is calculated from the title on the following manner:

  • remove first "static" word
  • add the method name
  • if there are parameters, then add "__" (2 underline chars)
  • add parameter names delimited by 1 underline char
  • remove spaces and punctuations from the result

For example: <@item title="static dobj(obj-opt, data_def-opt)"> will produce the following id: "dobj__obj_data_def"

You can use a @link keyword to create a hyperlink. For example, <a href="@link:/core/data-object.js/Data_Object.html#dobj__obj_data_def">Data_Object.dobj()</a> will produce the following result: Data_Object.dobj()

id (optional)

Id attribute for the generated HTML header element (overrides the id calculated from the title)

class (optional)

Class attribute for the generated HTML header element. Possible values: "overview" or "remark". Both produce a grayed header
