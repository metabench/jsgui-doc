<@overloads> tag

The <@overloads> tag generates an expandable block (initially collapsed) for method overloads.

Example:
<@codeblock> <@overloads title="field()"> <@item title="field()">

field() description

<@item title="field(field_name)">

field(field_name) description

<@item title="field(obj)">

field(obj) description

Result (live):
<@overloads title="field()"> <@item title="field()">

field() description

<@item title="field(field_name)">

field(field_name) description

<@item title="field(obj)">

field(obj) description

Attributes:
title

The overloads title, e.g. "field()". Usually the title is a method signature without parameters.

The <@overloads> tag adds an HTML id attribute to the generated header code. It allows to refer the item like <a href="filename.html#item_id">. The id is calculated like <@item> tag id, also adding "__" (2 underline chars) at the end.

For example: <@overloads title="field()"> will produce the following id: "field__"

You can use a @link keyword to create a hyperlink. For example, <a href="@link:/core/data-object.js/Data_Object.html#field__">Data_Object.field()</a> will produce the following result: Data_Object.field()
