GenDoc.exe command line

When GenDoc.exe launched without command-line arguments, it performs a full documentation rebuild, including the unit tests execution. But the tests execution spends many time. You can reduce the time using command-line arguments:

/skip-tests

The unit tests will be not executed at all. <@codeblock src="..."> tags will use the results from a previous tests execution. All the documentation will be rebuilt.

Example:
<@codeblock> GenDoc.exe /skip-tests
. (dot)

Will be executed the unit tests from the current directory and nested directories only. All the documentation will be rebuilt.

Example:
<@codeblock> GenDoc.exe .
(filename).spec.js

Will be executed the only unit test from the current directory with the specified file name. All the documentation will be rebuilt.

Example:
<@codeblock> GenDoc.exe Data_Object.spec.js
