<static> Class

Base class like John Resig's "Simple JavaScript Inheritance" tehnique.

In addition, when the inherited class contains a property that name starts from '#', then the property adds to the class without '#', and the property value is the base class property value, but the base class property name used is the rest of the initial property name (after the '#').

See:

  • John Resig's "Simple JavaScript Inheritance"

<static> clone(obj, count)

Creates a clone of the given value. The result depends of the input value type:

  • array: an array containing reference-copies of the input array elements
  • undefined: undefined
  • string: same string
  • object: deep copy of the object
  • other (number, boolean, null etc.): empty object (i.e. {})

If the second parameter is passed, then returns an array containing the requested number of the clones.

Parameters:
Name Type Argument Description
obj *

source value
count number <optional>

number of the output values
Example
clone("abc") ==> "abc"
clone("abc", 3) ==> ["abc", "abc", "abc"]

clone([1, undefined]) ==> [1, undefined]

clone({a: 1, b:{c:2}}) ==> {a: 1, b:{c:2}}

clone(1) ==> {}

<static> each(collection, fn, context)

Iterates over a collection of elements. Each element is passed to an iterator function. The iterator function is bound to the context object, if passed. The collection parameter can be:

  • a Collection object (the Collection.each() method is used in this case)
  • an array (iterate over array elements)
  • an object (iterate over object properties)

The iteration function parameters are: index, element, stopFunction (for array) key, value, stopFunction (for object)

The stopFunction breaks the iteration if called.

Parameters:
Name Type Argument Description
collection Collection | Array | Object

collection of elements
fn module:core/jsgui-lang-essentials.arrayIteratorCallback | module:core/jsgui-lang-essentials.objectIteratorCallback

iterator function
context Object <optional>

context object

<static> functional_polymorphism(options, fn)

Helps to create a polymorphic version of the passed function. The function will get 2 parameters: a and sig. Array a will contains the actual parameters, a.l property will contains the parameters count. The sig parameter will contains the parameter types array signature. The function will use these parameters to select a required polymorphic code branch to execute.

Parameters:
Name Type Argument Description
options * <optional>

not used
fn function

the function
Example
var myFunc = functional_polymorphism(function(a, sig){
   return stringify(a) + ": " + sig;
});

myFunc(1, "a", [null])  ==>  '[1, "a", [null]]: [n,s,a]'

<static> get_item_sig(i, arr_depth)

Returns a type signature for the given value. Possible results are:

  • "s" (string)
  • "n" (number)
  • "b" (boolean)
  • "f" (function)
  • "a" (array)
  • "o" (object)
  • "u" (undefined)
  • "!" (null)
  • "c" (control)
  • "r" (RegEx)
  • "B" (Buffer)
  • "R" (readable_stream)
  • "W" (writable_stream)
  • "X" (collection_index)
  • "D" (data_object)
  • "~D" (abstract data_object)
  • "V" (data_value)
  • "~V" (abstract data_value)
  • "C" (collection)
  • "~C" (abstract collection)
Parameters:
Name Type Argument Description
i *

value to get the type signature
arr_depth * <optional>

array nesting depth to include the array element signatures
Example
get_item_sig(1) ==> 'n'
get_item_sig("1") ==> 's'
get_item_sig([1, 2]) ==> 'a'
get_item_sig([1, 2], 1) ==> '[n,n]'

<static> jq_extend(deep, target, extender)

Merge the contents of two or more objects together into the first object. Returns the updated object. If only one parameter is passed, the extends this.

Parameters:
Name Type Argument Description
deep boolean <optional>

if true, performs recursive deep copy. if false or omitted, performs simple reference copy.
target object | array <optional>

target object; if omitted then this is used.
extender object | array <repeatable>

object(s) to add the content
Example
jq_extend({a:1}, {b:2})  ==> {a:1, b:2}
jq_extend([1,2,3], [4,5])  ==> [4,5,3]  

var ext = {sub2:{prop2:2}};
var result1 = jq_extend(true, {}, ext)  ==> {sub2:{prop2:2}}
var result2 = jq_extend(false, {}, ext)  ==> {sub2:{prop2:2}}
ext.sub2.prop2 = 20;  ==> (result1.sub2.prop2 == 2) & (result2.sub2.prop2 == 20)

<static> tof(obj, t1)

Returns the type of the given object as a string. If the t1 parameter is passed, then returns t1 for all the t1 values excepting 'object' one. Possible results are:

  • "number"
  • "string"
  • "function"
  • "boolean"
  • "object"
  • "undefined"
  • "null"
  • obj.__type
  • "array"
  • "control"
  • "regex" (obj instanceof RegExp)
  • "buffer" (obj instanceof Buffer)
  • "readable_stream" (obj instanceof Stream.Readable)
  • "writable_stream" (obj instanceof Stream.Writable)
Parameters:
Name Type Argument Description
obj *

object to inspect type
t1 string <optional>

type to return (?)
Example
tof(1) ==> "number"
tof([]) ==> "array"
tof(1, "myType") ==> "myType"
tof(1, "object") ==> "object"
tof([], "object") ==> "array"
